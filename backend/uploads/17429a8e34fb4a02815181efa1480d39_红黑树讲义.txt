红黑树（Red-Black Tree）是一种自平衡二叉搜索树，由 Rudolf Bayer 于 1972 年发明。

红黑树的五个性质：
1. 每个节点要么是红色，要么是黑色。
2. 根节点是黑色的。
3. 每个叶子节点（NIL）是黑色的。
4. 如果一个节点是红色的，则它的两个子节点都是黑色的。
5. 从任意节点到其每个叶子节点的所有路径都包含相同数量的黑色节点。

插入操作：
红黑树的插入操作时间复杂度为 O(log n)，其中 n 为树中节点数量。
插入新节点时，新节点初始为红色，然后通过旋转和重新着色来恢复红黑树的性质。
插入最多需要两次旋转操作。

删除操作：
删除操作时间复杂度也为 O(log n)。
删除节点后需要重新平衡，最多需要三次旋转。

应用场景：
- C++ STL 中的 map 和 set 容器底层实现
- Java 中的 TreeMap 和 TreeSet
- Linux 内核中的完全公平调度器（CFS）
- 数据库索引（如 InnoDB 的 B+ 树是基于 B 树的变体）

常见面试问题：
1. 红黑树和 AVL 树的区别是什么？
   AVL 树是严格平衡的（左右子树高度差不超过 1），红黑树是弱平衡的。
   AVL 树查找更快，但红黑树插入删除更快。

2. 红黑树的时间复杂度？
   查找、插入、删除均为 O(log n)。
